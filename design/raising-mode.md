## 0. 한 줄 요약 / 범위 경고

이 기능은 사실상 **클릭 불가(click-through) 데스크톱 오버레이 위에 얹는 미니 포켓몬 RPG**입니다.
배틀·경험치·진화·포획·파티·아이템·타입상성·기술학습을 모두 포함하므로, 앱의 규모가 크게 커집니다
(현재 `Sources/main.swift` 단일 파일 1,044줄 → 여러 모듈로 분할 필요, D20).

따라서 이 문서는 **① 요구사항 충돌 정리 → ② 데이터 감사(추가 추출) → ③ 결정 트리 → ④ 아키텍처 → ⑤ 단계별 로드맵** 순으로 정리합니다.

## 2. 데이터 현황 감사 (추가 추출해야 할 데이터)

### 2.1 이미 보유 (rom-extract `out/` + 번들 스프라이트)
- **스프라이트**: `animations/<id>/` — **Idle / Walk / Sleep** + 그림자 (1·2세대 251마리)
- **기술**: `moves.json` (559) — 위력·타입·분류(물리/특수/상태)·PP·명중률·크리티컬·사거리
- **학습셋**: `learnsets.json` — 레벨업(레벨+기술)/TM·HM/알
- **진화**: `evolutions.json` (259) — from→to·방법·조건(레벨/아이템/IQ/링크케이블/성별/리본 등)
- **레벨별 스탯**: `level_stats.json` — 570마리 × 100레벨 (HP/공/특공/방/특방 성장치 + 누적)
- **이펙트 스프라이트**: `effects/` — 기술 애니메이션(291개, 프레임 9,927장) + `move_animations.json`(기술→이펙트 매핑)
- **문자열**: 기술/몬스터/아이템/타입 이름·설명 (현재 언어키 `e`=영어)
- **base form 판별 가능**: `monsters.json`의 `pre_evo_index==0`

## 3. 설계 결정 트리 (각 항목 = 그릴 질문 + 추천)

### 로스터/시작
- **D1. 로스터 범위** — 1·2세대 라인이 3·4세대로 뻗는 경우 처리
  - 추천: **①1·2세대 + 진화완결용 3·4세대만 보강 다운로드**(~15–25마리). 진화가 끊기면 키우기 재미가 깨짐.
  - 대안: ②251 안에서 완결되는 라인만 / ③마지막 보유 스프라이트에서 진화 정지.
- **D17. 시작 조건** — #1·#2·#3
  - 추천: 스타터 후보 = `pre_evo_index==0`인 base form만. **레벨 5** 고정, 스탯은 `level_stats`의 L5 누적값.
    초기 기술 = level ≤5 level-up 기술 중 최신 4개. 성별은 대표 1종(G).
- **D16. 리셋** — #1: 세이브 삭제 → 스타터 선택부터. (확인 다이얼로그, 상세 GUI에서)

### 전투 핵심
- **D2. 배틀 애니 충실도**
  - 추천: **v1은 기존 Idle/Walk + 이펙트 스프라이트 + HP바**로 표현(전진→기술이펙트 재생→피격 시 깜빡/넉백). Attack/Hurt 포즈는 폴리시 단계(phase 2)에서 PMDCollab 추가 다운로드.
- **D3. 배틀 모델** — #8 자동 사용
  - 추천: **자동 턴제**. 속도(스탯 or 기술 speed) 순으로 번갈아, 기술은 간단 AI로 자동 선택(예: 최대 기대 데미지/상성 우선). 오버레이에서 관전만(#12).
- **D4. 규칙 충실도(C2)**
  - 추천: 스탯=`level_stats` 누적, 데미지=**단순화 본가식**(레벨·해당 스탯·기술위력·STAB·타입상성·소량 난수). **개체값/노력치/성격/특성 제외**.
- **D5. 타입 상성 출처(A)**
  - 추천: **EoS ROM 타입표 추출**(데이터 일관성). 여의치 않으면 표준 4세대 표 하드코딩.
- **D19. 상태이상** — #13에서 언급(상태/HP에 따른 포획률)
  - 추천: **v1은 최소화**(HP%만 포획률에 반영, 상태이상은 잠·마비 정도만 or 아예 제외 후 phase 2). 풀 상태이상 시스템은 범위 큼.
- **D10. 전투 결과/기절**
  - 추천: 액티브 기절 → 파티 내 다음 포켓몬으로 교체. 전멸 → 배틀 종료·시간에 따라 HP 회복(퍼머데스 없음). 승리 시 경험치 획득.

### 성장/진화
- **D6. 경험치·레벨업**
  - 추천: 승리 시 **패배한 야생 레벨 기반**으로 경험치 지급(`exp_required` 곡선 사용). 레벨 상한 100(또는 최종 진화까지).
- **D7. 기술 학습(4개 제한)** — #5, C1
  - 추천: 레벨업으로 새 기술 조건 충족 시, **상세 GUI 창**에 학습 프롬프트(배울지/어느 기술 대체할지). 오버레이 클릭 불가 유지.
- **D8. 진화 트리거(C3)** — #9
  - 추천: v1 **LEVEL 진화만 자동**(레벨 도달 시 진화 이펙트 후 진화). ITEMS/IQ/링크케이블 진화는 (a)제외 또는 (b)아이템/특수조건 후속. 진화 이펙트는 `effects/`에서 적절한 것 선정.

### 야생/아이템/포획
- **D9. 야생 출현 로직** — #4·#7
  - 추천: 불규칙 간격(랜덤 쿨다운)으로 1마리 스폰, 레벨은 플레이어 근처로 스케일, 로스터에서 추첨. **일정 시간 방치 시 소멸**(#7). 스폰 위치는 화면 가장자리.
- **D11. 포획** — #13
  - 추천: **몬스터볼 보유 시** 배틀 중 포획 시도(자동 or 조건). 확률 = 포획률(C)·HP%·(상태) 기반. 포획 성공 시 파티 편입.
- **D14. 파티/보관** — #14
  - 추천: **최대 6마리**. 6마리 시 포획하면 상세 GUI에서 "기존 1마리 방출 vs 신규 포기" 선택(C1). 방출은 영구(별도 보관함 없음, 단순화).
- **D12. 아이템** — #11·#13
  - 추천: 소수 큐레이션 — 몬스터볼(1~2종), 회복약(1~2종). 무작위로 오버레이에 스폰, **포켓몬이 지나가며 닿으면 획득**(자동, 클릭 불가). 볼=포획 재고, 회복약=HP 회복.

### UI/저장/전환
- **D15. 모드 전환 + 상세 UI** — #10, C1
  - 추천: 설정 창에 **일반/키우기 모드 토글**. 새 **상세 창**(클릭 가능)에 파티·스탯·경험치·기술·기술정보 표시 + 프롬프트(학습/방출) 호스팅. 오버레이 자체는 클릭 불가(#12).
- **D21. 저장(persistence)**
  - 추천: `Application Support`에 **JSON 세이브**(파티·레벨·경험치·기술·아이템·활성 포켓몬·모드). 재시작 후 이어서.
- **D22. 배틀 시각 디테일**
  - 추천: HP바(양쪽), 기술 이펙트 스프라이트 재생(보유), 피격 깜빡, 기절 페이드, 진화 이펙트. 텍스트 로그는 상세 창(선택).

## 4. 아키텍처 노트

- **파일 분할**: 현재 `Sources/main.swift` 1,044줄. `build.sh`가 단일 파일 컴파일이므로,
  여러 `.swift`를 컴파일하도록 `build.sh` 수정 후 모듈 분리 권장:
  - `Overlay/` (기존 SpriteView·CharacterController) — 오버레이는 **battle/pet 렌더 소비자**로 확장
  - `Raising/BattleEngine.swift` (턴제·데미지·상성·AI)
  - `Raising/PartyState.swift` (파티 6·스탯·경험치·기술·저장)
  - `Raising/WildSpawner.swift`, `Raising/ItemSpawner.swift`
  - `Raising/GameData.swift` (out/의 moves·evolutions·level_stats·typechart JSON을 번들해 로드)
  - `Raising/DetailWindow.swift` (상세 GUI + 프롬프트)
- **데이터 번들링**: `out/`의 필요한 JSON(가공·경량화본)을 앱 리소스로 번들. ROM/원본은 재배포 금지이므로
  **가공 결과 JSON만** 포함(스탯·기술·진화·타입표·매핑). 스프라이트는 기존 방식(PMDCollab).
- **루프 통합**: 기존 60fps 타이머(`setupTimer`)에 배틀/스폰/아이템 틱을 얹되, 배틀 중엔 follow 물리를 일시정지(#4).
- **오버레이 비클릭(#12)**: 현재 오버레이 윈도우는 이미 click-through. 상세 창만 일반 윈도우.

    **원본 데이터 보정 3가지**(그대로 쓰면 이상하게 보임): ① 파일 내 애니메이션 선택자는 **general 엔트리의 unk2**(2026-07-08 해독 — FILE1 unk2가 0~25를 1:1 열거, 가루 3종→0/1/2, 하품→거품 1개, 거품→거품 클러스터로 의미 검증. README의 옛 unk1 기록은 오류였음). ② 프레임이 큰 셀 안 비중심(48,159)에 있으므로 **union bbox 크롭 후 타깃 중심 정렬**. ③ 색: `approx_common` 팔레트는 **기술 타입색으로 재색조**(.color 블렌드, 거품 분홍→파랑), 공유파일 파티클(WAN_FILE0/1)은 프레임이 **복원 불가한 단색 사각형**(모양이 런타임 팔레트 그라디언트에 있었음)이라 프레임 크기·타이밍만 유지하고 **코드로 그린 둥근 글로우 입자**(노말=흰 섬광, 그 외 타입색)로 대체 후 7카피 링 버스트로 합성. 시각 검증: 거품(터지는 거품 2개)·불꽃세례(화염 혀)·10만볼트(낙뢰)·잎날리기(흩날리는 잎)·수면가루(낙하 가루)·물대포(물 스플래시).

  - ✅ **2i 진화 연출**: 진화 시(`.raisingEvolved`) 팔로워 위에 흰색 라디얼 버스트가 펄스하며 페이드(150틱). 스프라이트 교체를 커버. (풀 진화 시퀀스는 Phase 4 폴리시)
